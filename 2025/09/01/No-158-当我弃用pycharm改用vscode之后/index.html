No.158-当我弃用pycharm改用vscode之后

Hope
2025-09-01 (Updated: 2025-09-01)

导读

作为一名Python开发者，我使用PyCharm已经超过五年了。它无疑是一款强大的IDE，智能提示、调试功能、项目导航都无可挑剔。但随着时间的推移，一些不适开始浮现：

内存占用惊人：打开大型项目时，PyCharm动不动就吃掉4GB+内存，我的16GB笔记本开始捉襟见肘。尤其是使用AI 插件后，内存占用更是飙升。代码补全显得很卡顿，严重影响编码体验。而且我 256GB 的 SSD 硬盘也吃不消了。

启动速度慢：从点击图标到真正开始编码，等待时间比较久，打开 Pycharm，一般要去个卫生间再回来开始工作会比较顺畅。

过于“重量级”：有时候我只是想快速编辑一个小脚本，却需要启动整个IDE。单个文件编辑，其实 sublime text 3 就足够了。

尝试转变：初体验VSCode

一开始我是怀疑的——一个“编辑器”真能替代“IDE”吗？

安装必要的Python扩展后，我惊讶地发现VSCode提供了绝大多数我需要的功能：

  • 智能提示：通过Pylance扩展，代码补全几乎不输PyCharm
  • 调试支持：完整的断点调试、变量监视功能
  • 虚拟环境支持：轻松切换不同Python解释器
  • Git集成：源代码管理直观易用

绝了绝了。

我也是尝试了很多次，切换到VSCode后，感觉不顺再切换回来。

这样尝试了 7～8 次，我终于决定彻底抛弃PyCharm，拥抱VSCode。

惊喜发现：VSCode的独特优势

使用几周后，我发现了VSCode许多令人欣喜的特点：

闪电般的启动速度：几乎瞬间启动，随时记录灵感不再是问题

内存友好：通常内存占用只有PyCharm的1/3到1/2

无缝多语言支持：前端项目常常需要同时处理Python、JavaScript、HTML，VSCode天生就是多语言环境

扩展生态丰富：几乎任何需求都能找到对应的扩展，且安装管理极其简单

高度可定制：通过settings.json可以精细调整每一个细节

适应挑战：需要克服的障碍

转变并非一帆风顺，确实有一些需要适应的地方：

项目管理方式不同：VSCode以文件夹为基础而不是“项目”概念，需要时间适应

部分高级功能需要配置：一些PyCharm开箱即用的功能，在VSCode中需要额外设置

重构工具略逊一筹：虽然可用，但PyCharm的重构工具确实更强大

我的VSCode Python开发配置

经过不断调整，这是我的黄金配置：

必备扩展：

  • Python (Microsoft)
  • Pylance
  • Jupyter
  • Python Docstring Generator
  • Black Formatter
  • MagicPython
  • Python Debug (Microsoft)

结论：适合的才是最好的

经过几个月的使用，我确定VSCode已经满足了我95%的Python开发需求。剩下的5%主要是超大型项目的重构和深度数据库集成，这些场景下我仍然会打开PyCharm。

适合VSCode的场景：

  • Web开发（尤其是全栈开发）
  • 数据科学和脚本编写
  • 快速编辑和轻量级项目
  • 需要频繁切换语言的项目

可能仍需PyCharm的场景：

  • 超大型项目导航和重构
  • 复杂的数据库集成开发
  • 需要IDE级深度集成的企业环境

转变编辑器不是非此即彼的选择，而是找到最适合自己工作流程的工具。对我来说，VSCode提供了更好的平衡点：既强大又灵活，既功能丰富又保持轻量。

也许有一天我会再次改变我的工具选择，但至少现在，VSCode让我享受编码的乐趣变得更加简单直接。

安利一波：VSCode 的搜索功能是真的好用。
