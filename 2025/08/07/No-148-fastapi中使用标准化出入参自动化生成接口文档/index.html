No.148-fastapi中使用标准化出入参自动化生成接口文档

Hope
2025-08-07 (Updated: 2025-08-14)
fastapi

导读

在 fastapi 中，自动化生成文档是一个非常简单的事情。

但是也需要遵循一些非常必要的规则来帮助我们更好的生成便于阅读的文档。

这篇文章将介绍从入参的定义、到返回结构体的定义，异常捕获几个方面来规范接口开发文档。

使用的参数校验库为：pydantic。

入参的定义

必填参数

说明：

  • 定义 class 类
  • 继承 BaseModel
  • 使用 Field 定义参数类型等。
  • 这里定义了service_id字段为int类型
  • Field 第一个参数的默认值为 ...，就表示是必填参数

非必填参数

说明：

  • 这里定义了order_id字段为int类型
  • 使用 Optional 包裹住 int 表示非必填
  • 结合Field 第一个参数的默认值为 default=None，就表示是非必填参数

注意：这里的 default=None 不能简写成 None，简写之后文档将识别不了，不知道这个问题 fastapi 官方修复了没。

字符串参数长度定义

说明：

  • 这里定义了env_id字段为str类型
  • 使用 Optional 包裹住 str 表示非必填
  • 结合Field 第一个参数的默认值为 default=None，就表示是非必填参数
  • max_length 定义字段最长为 8 个字符
  • min_length 定义字段最短为 4 个字符
