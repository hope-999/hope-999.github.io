优化版 Dockerfile（支持 ASGI + 多阶段构建）

关键优化点解析

  1. ASGI 服务器选择

    • 使用 UvicornWorker + Gunicorn 组合：
      • Uvicorn 处理异步请求
      • Gunicorn 管理进程（建议 worker 数 = min(2*CPU核心+1, 8)）

  2. 安全增强

    • 禁用 root 用户运行容器
    • 通过 .dockerignore 排除 __pycache__、.env 等敏感文件
    • 使用 --no-warn-script-location 避免权限警告

  3. Kubernetes 适配

    • 健康检查：在 Deployment 中配置：

    • 资源限制：建议在 K8s 中设置 CPU/Memory 限制

    • 环境变量：敏感配置通过 K8s Secrets 注入

  4. 性能优化

    • 多阶段构建减少镜像体积（最终镜像约 150MB）
    • 依赖分层安装（修改代码时不会重新安装依赖）

不同场景的启动命令调整

场景 CMD 指令示例
纯开发环境 CMD ["uvicorn", "app.main:app", "--reload", "--host", "0.0.0.0"]
高并发生产环境 CMD ["gunicorn", "-k", "uvicorn.workers.UvicornWorker", "--bind", "..."]
单节点测试 CMD ["uvicorn", "app.main:app", "--workers", "4"]
