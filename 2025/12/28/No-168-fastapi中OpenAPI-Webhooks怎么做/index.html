No.168-fastapi中OpenAPI-`Webhooks`怎么做

Hope
2025-12-28 (Updated: 2025-12-28)

导读

大家好！今天来聊聊 FastAPI 中的 OpenAPI Webhooks。如果你正在开发需要向第三方推送消息的服务，比如订单状态变更、用户注册成功等场景，那 Webhooks 绝对是你的好帮手。

什么是 Webhooks？简单理解就是”反向 API”：不是别人调你，而是你主动推送消息给别人。就像外卖小哥给你打电话通知”你的外卖到了”一样。

方案

为什么需要 Webhooks？

想象一下这个场景：用户在你的平台上完成了支付，你需要立即通知第三方系统更新订单状态。如果用传统的 API，第三方需要不停地轮询你的接口：”订单支付了吗？”、”订单支付了吗？”……这样既浪费资源，实时性也差。

而 Webhooks 则是”主动推送”模式：支付成功了？好，我马上通知你！这样既高效又及时。

自动生成 API 文档

FastAPI 最棒的地方在于，它会把你的 Webhooks 自动加入到 OpenAPI 文档中。启动应用后访问 http://127.0.0.1:8000/docs，你就能看到：

  • Webhook 的名称
  • 请求体的数据结构
  • 字段说明和类型

这样，第三方系统开发人员就能清楚地知道应该如何处理你发送的 Webhook 数据了！

注意事项

  1. 错误处理：Webhook 接收方可能暂时不可用，要做好重试机制和超时处理
  2. 安全性：建议添加签名验证，确保 Webhook 请求来自你的系统
  3. 幂等性：接收方应该能够安全地处理重复的 Webhook 请求

总结

FastAPI 的 Webhooks 功能让”主动推送消息”变得简单优雅。通过 app.Webhooks.post() 定义事件，配合 Pydantic 模型定义数据结构，就能生成清晰的 API 文档。

无论你是要通知第三方系统订单状态，还是推送用户注册事件，Webhooks 都是最佳选择。记住要做好错误处理和安全性验证，这样你的系统就能稳定可靠地运行了！
